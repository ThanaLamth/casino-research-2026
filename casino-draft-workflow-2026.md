# Casino Draft Workflow 2026

This file is the current working workflow distilled from the full June 2026 session.

Its purpose is simple:

- create a strong draft ready to publish before manual image collection
- keep the article source-led and evidence-disciplined
- collect images only after the draft structure is already locked
- output a separate image shot list so editors know exactly what to capture

## Core Principle

The article comes first.

Do not block drafting on screenshots.

The correct order is:

1. choose the page angle and market scope
2. write the draft to a publish-ready standard using source-led evidence
3. generate a separate manual image shot list
4. collect only high-value images
5. patch the images back into the article

## Final Workflow

## Step 1. Lock the Page Type

Choose one page type before writing:

- single-brand review
- top-list comparison page
- market page

Do not mix regulated US, offshore, and crypto brands in the same page unless the article is explicitly a market-split comparison.

## Step 2. Lock the Evidence Standard

The draft should be:

- source-led
- compliance-aware
- explicit about what was and was not verified first-hand

Do not claim:

- funded deposit testing if none happened
- verified cashout speed if none was tested
- logged-in proof if the asset is only public-page evidence
- support quality if there was no real support interaction

Use neutral editorial language such as:

- we reviewed
- based on available evidence
- current evidence suggests
- public materials indicate
- this remains unverified

## Step 3. Build the Draft First

The first output should always be a draft-ready article, not an image collection task.

That draft should already be close to publishable and include:

- meta title
- meta description
- excerpt
- slug
- primary category
- secondary category
- suggested tags
- disclaimer
- direct-answer intro
- trust or methodology box
- comparison table when relevant
- full review sections
- responsible gambling section
- FAQ
- final verdict

This draft should already read like a real article even before the missing images are collected.

## Step 4. Treat Images as Evidence, Not Decoration

After the draft is complete, create a separate file listing the exact manual images still needed.

Only request high-value images.

High-value images are:

1. logged-in cashier, deposit, or withdrawal page
2. live support screenshot with a real reply
3. signup or onboarding flow with meaningful friction
4. logged-in dashboard or account area
5. licensing, responsible gambling, or compliance proof
6. real-world device photo

Low-value images are:

- generic homepages
- hero banners
- stock-looking screenshots that add no new claim
- mockups presented as if they were review evidence

## Step 5. Output Two Files

Every serious article run should produce exactly two main outputs:

### Output A. Publish-Ready Draft

This is the full article draft.

It should:

- be complete enough to publish after image upgrades
- already include any currently available real evidence
- clearly label public-page images versus stronger first-hand assets
- avoid overclaiming

### Output B. Manual Image Shot List

This is the editor task file.

It should include for every requested image:

- brand
- image type
- priority
- exact article placement
- filename
- what to capture
- why it matters
- alt text
- caption
- status

This second file exists so editors can collect assets efficiently without guessing what the article still needs.

## Step 6. Only Patch Images Back After Collection

Do not delay writing while waiting for images.

Once manual assets are collected:

- replace weak public-page images first
- upgrade support entry-state images to real reply images
- add cashier proof to the strongest sections
- convert approved files to webp when practical
- keep captions and alt text precise

## Article Standard

The target is not a rough outline.

The target is a publish-ready source-led draft with clear evidence gaps.

That means:

- the article should be readable now
- the article should improve further once manual images are inserted
- the image file should function like a production checklist

## Working Rules from This Session

These rules are now part of the workflow:

- prioritize high-value images only
- draft first, collect images second
- output a separate image-list file for manual collection
- use captions and alt text for every image
- when a full article is produced, include excerpt, meta description, meta title, and disclaimer
- do not present fake first-hand evidence
- do not use generic screenshots just to make the article look fuller
- if an image is only public-page evidence, say so directly

## Current Recommended Deliverable Pattern

For each new casino page or market page, produce:

1. one publish-ready markdown article
2. one manual image shot list markdown file

That is now the default workflow to follow.
